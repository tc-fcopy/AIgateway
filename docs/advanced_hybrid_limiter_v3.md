# 高级混合限流中间件v3使用说明

## 概述

高级混合限流中间件v3 (`http_flow_limit_v3.go`) 是一个基于内存和Redis的混合限流解决方案，旨在提供高性能、可扩展的限流功能。它结合了内存限流的高速度和Redis分布式限流的一致性。

## 特性

1. **内存限流**：使用分片令牌桶算法实现高性能的内存限流
2. **Redis分布式限流**：通过Redis实现跨节点的全局限流
3. **服务级限流**：针对特定服务的高并发限流
4. **用户级限流**：针对特定用户的精准限流
5. **自适应调整**：根据系统负载动态调整限流策略

## 实现原理

### 内存限流组件

- 使用分片令牌桶算法减少锁竞争
- 通过哈希算法将请求分散到不同的分片桶中
- 支持动态调整令牌补充速率

### Redis分布式限流组件

- 使用Redis存储用户令牌桶状态
- 通过Lua脚本实现原子性的滑动窗口计数
- 支持批量同步减少网络开销

### 限流策略

1. **服务级限流**：
   - 使用分片令牌桶算法
   - 通过哈希算法分散请求到不同分片
   - 支持动态调整分片数量

2. **用户级限流**：
   - 使用LRU缓存管理用户令牌桶
   - 定期同步状态到Redis
   - 支持最大用户桶数量限制

## 配置要求

确保在配置文件中正确配置Redis连接信息：

```toml
# conf/dev/redis_map.toml
[list]
    [list.default]
        proxy_list = ["127.0.0.1:6379"]
        conn_timeout = 1000
        read_timeout = 1000
        write_timeout = 1000
        max_active = 100
        max_idle = 100
        idle_timeout = 300
```

## 使用方法

在路由中添加中间件：

```go
r.Use(http_proxy_middleware.HTTPFlowLimitMiddlewareV3())
```

## 性能优化

1. **分片策略**：根据CPU核心数自动调整分片数量
2. **批量同步**：减少Redis同步频率
3. **异步处理**：避免阻塞主流程
4. **连接池管理**：复用Redis连接

## 监控指标

- 服务级限流通过率
- 用户级限流通过率
- Redis同步延迟
- 内存使用情况

## 故障处理

1. Redis连接失败时降级为纯内存限流
2. 自动清理非活跃用户桶
3. LRU算法防止内存溢出
